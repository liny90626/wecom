# WeCom Changelog Index

本目录记录 OpenClaw WeCom 插件的版本变更。默认先看本 fork 最近 5 个维护版本；原仓库历史版本保留在后续列表中，便于回溯。

## 本 fork 最近 5 个维护版本

- [`v2.5.110-144`](./v2.5.110-144.md)（发布标签 `released/2.5.110-144`）：外部 source reply 已确认投递后，旧 Bot WS 流直接静默结算，不再误启动 final retry 或推送“本次回复投递中断”；没有新增状态或 timer。
- [`v2.5.110-143`](./v2.5.110-143.md)（发布标签 `released/2.5.110-143`）：暖会话不再等待 OpenClaw best-effort metadata 队列，冷会话只保留 1 秒初始化窗口；错误 final 不再伪装成成功续文或追加完成标记，并修复生产依赖高危漏洞。
- [`v2.5.110-142`](./v2.5.110-142.md)（发布标签 `released/2.5.110-142`）：解除 reasoning/Fast 进度 ACK 对 OpenClaw 模型流的反向阻塞，串行合并进度并在 final 前有界封口；修正长任务 `0s` 和计时重置，保留 Fast 与真实模型超时语义。
- [`v2.5.110-141`](./v2.5.110-141.md)（发布标签 `released/2.5.110-141`）：汇总未发布 v140 体验线；OpenClaw routed final 主动推送成功后显式收口原 Bot WS 气泡，长任务满 9 分钟后按分钟更新实际用时，长文本分段更紧凑，原始失败 final 保持交付，并升级企微 SDK 1.0.7。
- [`v2.5.110-139`](./v2.5.110-139.md)（发布标签 `released/2.5.110-139`）：适配 OpenClaw 2026.7.1——繁忙会话消息被并入/排队、回合 yield 转后台不再误报“投递中断”，失败提示不再泄露思考内容；流窗过期后 final 结果不明失败按分片进度有界补发、耗尽推送失败提示；修复 v138 孤儿 drain 竞态；后台处理提示延至任务满 9 分钟。

> B1 Markdown 表格、B2 长文本分段/去重、B3 长任务与新消息合并等较早 fork 修复，已统一归档在 [`v2.5.110-112`](./v2.5.110-112.md)；reasoning 思考块系列修复见 [`v2.5.110-113`](./v2.5.110-113.md)。
