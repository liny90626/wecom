# OpenClaw WeCom 插件 v2.5.110-113 变更简报

> [!NOTE]
> `v2.5.110-113` 是 LinKy fork 维护版。本版本聚焦 reasoning 思考块实验的稳定性修复：OpenClaw 可能多次输出思考流和正文，且后续思考片段有时会以普通 block/final 文本中的 `<think>...</think>` 形式到达插件。

## 2026-06-27（v2.5.110-113）

### 修复

- 修复普通 block 文本中混入 `<think>...</think>` 时，后续思考内容被当成正文展示的问题。
- 修复 final 文本中混入 `<think>...</think>` 时，思考内容可能无法进入思考块或污染最终正文的问题。
- 抽取出的 thinking 会合并进现有思考块；标签外文本才进入正文预览或最终正文。
- final 中抽出的最后一段 thinking 会在 final 正文关闭 stream 前强制刷新一次进度流，避免“被抽走但没展示”的情况。

### 保护

- 行内代码和 fenced code block 中的字面量 `<think>...</think>` 不会被当成思考块抽取，仍按普通正文转义展示。
- 主动推送、长文本余段和 stream fallback 仍不会携带 `<think>` 思考块，避免思考内容泄漏到最终正文或主动推送正文里。
- B1 Markdown 表格、B2 长文本分段/去重、B3 新消息合并和长任务兜底路径保持不变。

### 回归测试

- 新增普通 block 中 inline think block 抽取测试。
- 新增 final 中 inline think block 抽取测试。
- 新增代码文本中 literal think tag 保护测试。
