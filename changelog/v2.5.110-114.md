# OpenClaw WeCom 插件 v2.5.110-114 变更简报

> [!NOTE]
> `v2.5.110-114` 是 LinKy fork 维护版。本版本聚焦 thinking 思考块与 B2 长文本分段同时启用后的投递体验修复。

## 2026-06-27（v2.5.110-114）

### 修复

- 修复 block/preview 阶段复用正式分段逻辑导致的“只显示 `【第1/n段】`，但后续段并不会立刻出现”的伪分段问题。
- 修复 thinking block 较长时，正文预览字符预算被过度压缩，导致正文首段很短就显示分段提示的问题。
- 修复旧分段逻辑先切正文、再追加长段标时可能二次截断并丢尾部文本的问题。
- 长文本分段提示从 `【消息过长，分段发送：第x/n段】` 简化为 `【第x/n段】`。
- 长文本完成标记统一追加到最后一段段标之后，即 `【第n/n段】` 后再显示 `（已完成）`。

### 行为说明

- 非 final 的预览消息不再显示分段标签；插件在预览阶段没有完整正文，不能可靠发送第 2/3 段。
- final 正文完整到达后，首段仍通过原 stream 气泡关闭，后续段通过主动 markdown 气泡继续发送。
- 后续主动分段仍不携带 `<think>`，避免思考内容在普通正文气泡中泄漏或被企业微信误渲染。
- B1 Markdown 表格、B2 final 去重、B3 新消息合并和 stream 过期兜底路径保持兼容。
