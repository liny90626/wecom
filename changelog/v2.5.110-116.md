# OpenClaw WeCom 插件 v2.5.110-116 变更简报

> [!NOTE]
> `v2.5.110-116` 是 LinKy fork 维护版。本版本聚焦 Bot WS 回复可靠性，不改变 B1 Markdown 表格、B2 长文本切分算法和 reasoning 思考块正文拆分规则。

## 2026-06-28（v2.5.110-116）

### 修复

- 为 Bot WS `replyStream` / 主动 markdown 推送增加本地发送超时保护。
- 当中间预览 stream 更新偶发长期 pending 时，插件会标记当前 stream 不可靠，避免后续 block/final 一直卡在同一个出站调用上。
- final 到达后若原 stream 已不可靠，会走主动 markdown 兜底；若前半段已经成功显示，则优先只推送“继续输出”部分，降低重复正文风险。

### 体验调整

- 长文本、长任务和 reasoning-only 场景末尾的完成标识从 `（已完成）` 调整为 `（回复完毕）`，避免和任务执行结果状态混淆。

npx vitest run src/transport/bot-ws/reply.test.ts
