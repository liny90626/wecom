# OpenClaw WeCom 插件 v2.5.110-117 变更简报

> [!NOTE]
> `v2.5.110-117` 是 LinKy fork 维护版。本版本基于 `2.5.110-116` 继续修复 Bot WS 偶发 stream ACK 队列阻塞问题；`released/2.5.110-116` 保留为上一稳定追溯点。

## 2026-06-29（v2.5.110-117）

### 修复

- 非 final 的 Bot WS 预览更新优先使用 SDK `replyStreamNonBlocking()`，当同一 `req_id` 已有未完成 ACK 时跳过本次中间帧，避免 stream preview 在 SDK 队列里持续堆积。
- final 到来时会检查同一 `req_id` 是否仍有 pending ACK：若 ACK 在短暂保护窗口内清除，仍走原 stream 气泡正常收尾；若持续未清除，则改走主动 markdown 兜底，避免 final 被排进旧 stream 队列后只能等下一条用户消息触发刷新。
- 保留 v116 的 stream 不可靠兜底和 `（回复完毕）` 标识，不改变 B1 Markdown 表格、B2 长文本分段/去重、B3 supersede 合并规则。

### 回归测试

- 新增 pending ACK 场景测试：确认预览更新被跳过且 final 不再进入旧 stream 队列，而是主动推送“继续输出”剩余内容。
- 新增 ACK 快速恢复场景测试：确认正常情况下仍使用原气泡 `replyStream(..., finish=true)` 收尾，不会过度 fallback 成新气泡。
